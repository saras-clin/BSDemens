### Socioeconomic Status (SES)
# Extracted separately in: 03_extract_ses.R
# Per SEPLINE guidelines (Hjorth et al. 2025, doi:10.2147/CLEP.S520772)
#
#   education_cat: HFAUDD from UDDA (highest completed education by year before surgery)
#     - "Short":   HFAUDD starts with "10" (primary) or "15" (preparatory)
#     - "Medium":  HFAUDD starts with "20" (general upper secondary), "30" (vocational), "35" (qualifying)
#     - "Long":    HFAUDD starts with "40"-"80" (tertiary and above)
#     - "Unknown": HFAUDD starts with "90" or missing
#
#   income_cat: FAMAEKVIVADISP_13 from FAIK (equivalized disposable household income)
#     - Quintile-based, age/sex standardized: "Low" (Q1), "Medium" (Q2-Q4), "High" (Q5)
#     - Linked via FAMILIE_ID in BEF to get household income per person
#
#   occupation_cat: SOCIO13 from AKM (employment classification, year before surgery)
#     - "Working":           codes 110-114, 120, 131-135, 139 (self-employed, managers, employees)
#     - "Student":           310
#     - "Unemployed":        210, 410
#     - "Outside_workforce": 220, 321, 330 (sick leave, disability, social security/flex job)
#     - "Retired":           322, 323
#     - "Unknown":           0, 420 (children), or missing

### Baseline Medications (ATC codes from LMDB, 5 years before surgery)
# Purpose:
#   (1) Required component of NMI/GMC conditions that need BOTH diagnosis AND medication
#       (e.g. treated hypertension = I10-I15 OR >=2 antihypertensive prescriptions/year)
#   (2) Independent covariates reflecting metabolic/psychiatric status at baseline
#
#   antihypertensive: C02, C03, C07, C08, C09
#   lipid_lowering:   C10
#   antidiabetic:     A10 (all); A10A = insulin only
#   antidepressant:   N06A
#   antidementia:     N06D (sensitivity check - flag persons already treated for dementia)

### Obesity-related comorbidities (from LPR diagnoses + LMDB, 5 years before surgery)
# Purpose: These are the primary BS eligibility criteria (BMI >=40, or >=35 + comorbidity)
#   and are key confounders in the BS-dementia association. They overlap with NMI conditions
#   intentionally - flagged separately because they directly determine BS eligibility.
#
#   hypertension:  I10-I15 OR antihypertensives (C02, C03, C07-C09)
#   diabetes_t2:   E11 OR non-insulin antidiabetics (A10B, A10X)
#   dyslipidemia:  E78 OR lipid-lowering (C10)

## PRIMARY OUTCOMES - WEIGHT ----
# Source: DBSO (confirm exact DST folder name with data manager)
# Variables:
#   weight_preop:      weight (kg) at last preoperative exam
#   height_preop:      height (cm) at preoperative exam
#   bmi_preop:         BMI at preoperative exam
#   weight_3mo/6mo/12mo/24mo: weight at follow-up visits
#   bmi_3mo/6mo/12mo/24mo:    BMI at follow-up visits
# Calculated:
#   twl:      total weight loss = weight_preop - weight_postop
#   pct_twl:  (twl / weight_preop) * 100
#   excess_weight: weight_preop - (25 * height_m^2)   [target BMI = 25]
#   pct_ewl:  (weight_loss / excess_weight) * 100

# DATA SOURCES MAPPING - DST parquet folder names

# Base path: E:/workdata/708421/cleaned-data/parquet-registers/
#
# BEF - Befolkningsregisteret (Population register)
#   DST folder: bef
#   Key vars (lowercase after rename_with(tolower)):
#     pnr, koen (sex: 1=M, 2=F), foed_dag (birth date), aar (year),
#     familie_id (for income linkage), reg (region), civst (marital status)
#   Note: does NOT contain death date - use DOD for that
#
# DOD - Doedsregisteret (Death register)
#   DST folder: dod
#   Key vars: pnr, doddato (date of death)
#
# LPR2 - Landspatientregisteret (National Patient Registry, 1977 to March 2019)
#   DST folders: lpr_adm (contacts) + lpr_diag (diagnoses)
#   lpr_adm:  pnr, recnum, d_inddto (contact date), c_spec (specialty), aar
#   lpr_diag: recnum, c_diagtype (A=primary, B=secondary, G=other), c_diag (ICD with D-prefix)
#   Join: lpr_adm LEFT JOIN lpr_diag ON recnum
#   ICD note: codes have "D" prefix (e.g. "DG30") -> use substr(c_diag, 2, 4) to get "G30"
#   Keep diagtypes: c("A", "B") -- exclude H (referral) and M (complications)
#
# LPR3 - Landspatientregisteret v3 (from March 2019 onwards, unified somatic + psychiatric)
#   DST folders: kontakter (contacts) + diagnoser (diagnoses)
#   (confirmed from hospital2021.R reference; "lpr3f_kontakter"/"lpr3f_diagnoser" returned 404)
#   kontakter:  pnr (raw SAS: cpr - dstDataPrep renames to pnr),
#               dw_ek_kontakt, dato_start (contact datetime), dato_slut,
#               hoofdspeciale_ans (specialty), kontakttype
#   diagnoser:  dw_ek_kontakt, diagnosekode (ICD with D-prefix), diagnosetype (A/B),
#               senare_afkraeftet ("Ja"/"Nej" - exclude "Ja" = later retracted),
#               diagnosekode_parent
#   Join: kontakter LEFT JOIN diagnoser ON dw_ek_kontakt
#
# Psychiatric LPR contacts
#   For LPR3: fully unified - psychiatric contacts included in kontakter;
#             all A/B diagnoses searchable without specialty filter.
#   For LPR2: t_psyk_adm + t_psyk_diag (pnr, recnum, d_inddto / recnum, c_diag, c_diagtype)
#             ** CONFIRM-6: ask data manager if load_database() or SAS file on project 708421 **
#
# LMDB - Lagemiddeldatabasen (National Prescription Database)
#   DST folder: lmdb
#   Key vars: pnr, atc (ATC code), eksd (dispensing date), indo (indication code)
#   Covers: 1994 onwards, annual files combined in parquet
#
# UDDA - Uddannelsesregisteret (Education register)
#   DST folder: udda
#   Key vars: pnr, hfaudd (highest completed education code), aar
#   Use: filter aar == year(surgery_date) - 1 to get education at baseline
#
# FAIK - Familieindkomst (Family income register)
#   DST folder: faik
#   Key vars: familie_id, famaekvivadisp_13 (equivalized disposable household income), aar
#   Link: join bef (pnr, familie_id, aar) with faik (familie_id, aar) to get income per person
#
# AKM - Arbejdsklassifikationsmodulet (Employment classification)
#   DST folder: akm
#   Key vars: pnr, socio13 (socioeconomic group code), aar
#   Use: filter aar == year(surgery_date) - 1
#
# DBSO - Danish Obesity Surgery Registry (Databasen for Behandling af Svær Overvægt)
#   Source: SAS file from SunDK (dfr_2025_10_31.sas7bdat); converted to parquet by 00_prepare_dbso.R
#   Parquet path: E:/workdata/708421/cleaned-data/parquet-external/databasesvaerovervaegt/
#   Access: arrow::open_dataset(path_dbso) — NOT via load_database()
#   Format: LONG — one row per clinic visit per patient (PRE=pre-op, PER=surgery, FOL=follow-up)
#   Note: mandatory reporting since 2010, covers all public and private bariatric procedures
#   All column names below are lowercase (rename_with(tolower) applied in 00_prepare_dbso.R)
#
#   --- PATIENT IDENTIFIER ---
#   cpr -> pnr        encrypted CPR number (renamed to pnr for consistency across pipeline)
#
#   --- SURGERY TYPE FLAGS (binary 0/1; patient-level: same value across all rows per patient) ---
#   gastricbypass_prim    1 = primary Roux-en-Y gastric bypass (RYGB)
#   gastricsleeve_prim    1 = primary sleeve gastrectomy (SG)
#   redo_prim             1 = revision/redo bariatric surgery of any type
#                             includes: revision bypass (gastricbypass_prim=1 AND redo_prim=1)
#                                       revision sleeve (gastricsleeve_prim=1 AND redo_prim=1)
#   gastricsleeveredo     1 = revision sleeve gastrectomy (sleeve-specific revision flag)
#   konverteret           1 = conversion procedure (e.g., SG converted to RYGB)
#
#   --- DERIVED COLUMN (added by 00_prepare_dbso.R) ---
#   surgery_type          "RYGB" / "SG" / "ReDo" / "Unknown"
#                         derived from flags above; redo_prim checked FIRST so revision bypass
#                         is classified "ReDo" not "RYGB" (case_when priority fix)
#
#   --- VISIT DATES ---
#   datopre               date of pre-operative assessment visit (DatoPRE)
#   datoper_prim          date of primary bariatric surgery — DBSO clinical record (DatoPER_prim)
#                         used as index date (surgery_date); confirmed as preferred over OpDateLPR
#   opdatelpr             surgery date as recorded in LPR / D_ODTO; used only for data quality
#                         check (DatoPER_prim vs OpDateLPR discrepancy) in explore_dbso()
#   datofol               date of this follow-up clinic visit; varies across rows per patient
#
#   --- FOLLOW-UP WINDOW FLAGS (binary 0/1; visit-level) ---
#   fumellem6mdr1aar      1 = this follow-up visit falls in the 6-month to 1-year window
#   fumellem1_5aar2_5aar  1 = this follow-up visit falls in the 1.5-year to 2.5-year window
#   id_funaermest2aar     row identifier for the follow-up visit nearest to 2 years post-surgery
#
#   --- WEIGHT AND HEIGHT (continuous; some columns patient-level, vaegtfol is visit-level) ---
#   udgangsvaegt          weight (kg) at program entry / referral (Udgangsvaegt)
#   udgangsvaegtpre_prim  weight (kg) at last pre-operative assessment (UdgangsvaegtPRE_prim)
#                         patient-level constant; used to compute bmi_pre
#   vaegtper_prim         weight (kg) at surgery day (VaegtPER_prim); patient-level
#   vaegtfol              weight (kg) at THIS follow-up visit; varies across rows
#   hoejde                height (cm); patient-level constant across all visit rows (Hoejde)
#   bmi_pre               derived: udgangsvaegtpre_prim / (hoejde/100)^2; added by 00_prepare_dbso.R
#
#   --- DBSO-COMPUTED CLINICAL OUTCOMES (Study 2; patient-level flags) ---
#   akutgenindlaeggelse30d  1 = acute unplanned readmission within 30 days of surgery
#   reop30d_3               1 = reoperation within 30 days
#   reoplaar_3              1 = reoperation within 1 year
#   reop5aar_2              1 = reoperation within 5 years
#   medkomplik*             5 medical complication flag columns — exact names must be
#                           confirmed by running inspect_dbso() on DST (run names(raw))
#                           Expected pattern: medkomplik, medkompliktype, medkomplikdato, etc.
#
#   --- COMORBIDITIES AND STAGING (Study 2; patient-level) ---
#   eoss                    Edmonton Obesity Staging System score at pre-op assessment (0-4)
#   obstruktivsoevnapnoepre  1 = obstructive sleep apnea documented at pre-op
#   obstruktivsoevnapnoefol  1 = obstructive sleep apnea documented at follow-up
#
#   --- NUTRITIONAL SUPPLEMENTS (Study 2; patient-level or visit-level — confirm on DST) ---
#   b12                     vitamin B12 supplement status or compliance flag
#   jern                    iron supplement status or compliance flag
#   kalk                    calcium supplement status or compliance flag
#                           Compliance sub-columns likely suffixed _compliance or similar;
#                           exact names must be confirmed by inspect_dbso() on DST
#
#   --- VARIABLE COVERAGE BY STUDY ---
#   Study 1 (Dementia):   surgery_date, surgery_type (exposure definition only)
#                         bmi_pre from extract_weights.rds (Table 1 descriptive)
#   Study 2 (T1D):        all weight/height columns (extract_weights.rds)
#                         akutgenindlaeggelse30d, reop30d_3, reoplaar_3, reop5aar_2 (extract_dbso_clinical.rds)
#                         eoss, obstruktivsoevnapnoepre/fol (extract_dbso_clinical.rds)
#                         b12, jern, kalk (extract_dbso_clinical.rds)
#                         medkomplik* (extract_dbso_clinical.rds — confirm column names first)
#
# OSDC - Open Source Diabetes Classifier (pre-computed cohort file)
#   File path: confirm with data manager (likely dm_population_YYYY_YYYY.rds)
#   Key vars: pnr (or PNR), diabetes_type (1=T1D, 2=T2D), do_dm (classification date)
#   Package: https://steno-aarhus.github.io/osdc/index.html
